\section{Executive Summary}

This document presents a comprehensive implementation plan for the development of an Augmented Reality (AR) system designed to enhance quality inspection processes for formed parts in manufacturing environments. The project is a collaboration between Hwashin Co., Ltd (Korea) and the Institute of Energy Technology at Hanoi University of Science and Technology (Vietnam).

\subsection{Project Overview}
\begin{itemize}
    \item \textbf{Project Title:} Development of AR System Technology for Quality Information Display
    \item \textbf{Total Budget:} \$20,000 USD
    \item \textbf{Duration:} August 15, 2025 - August 31, 2026 (12.5 months)
    \item \textbf{Primary Technology:} LiDAR-based AR on iOS devices (iPad/iPhone)
    \item \textbf{Project Manager:} Professor Nguyen Duc Toan (IET)
    \item \textbf{Responsible Manager:} Team Leader Yeo In-Joo (HWASHIN)
\end{itemize}

\subsection{Strategic Importance}
Augmented Reality technology offers significant potential to improve the accuracy and efficiency of visual inspection for manufactured parts. This project addresses the critical need for real-time quality assessment by developing an innovative mobile application that leverages LiDAR scanning capabilities to recognize objects and overlay quality data directly onto physical parts.

\section{Project Background and Motivation}

\subsection{Industry Context}
In modern manufacturing, particularly in the automotive and metal forming industries, quality inspection of formed parts is a critical yet time-consuming process. Traditional inspection methods involve:
\begin{itemize}
    \item Manual measurements with calipers and gauges
    \item Time-intensive coordinate measuring machine (CMM) operations
    \item Limited real-time feedback to production operators
    \item Difficulty in visualizing dimensional deviations
\end{itemize}

\subsection{Technology Gap}
Current inspection systems lack:
\begin{itemize}
    \item Mobile, on-site inspection capabilities
    \item Intuitive visualization of quality deviations
    \item Real-time comparison with design specifications
    \item Integration of 3D scanning with quality data mapping
\end{itemize}

\subsection{Solution Approach}
This project develops a mobile AR application that:
\begin{enumerate}
    \item Utilizes iPad/iPhone LiDAR scanners for 3D object recognition
    \item Imports template databases containing design specifications
    \item Compares scanned geometry with CAD data
    \item Visualizes dimensional differences using color-coded heat maps
    \item Provides immediate feedback to quality inspectors
\end{enumerate}

\subsection{Performance Specifications}
\begin{itemize}
    \item \textbf{Recognition Time:} $<$ 3 seconds per object

\section{System Architecture}

\subsection{Overall Architecture}
The system follows a modular architecture with the following layers:

\begin{tcolorbox}[colback=blue!5!white,colframe=blue!75!black,title=System Architecture Layers]
\textbf{1. Presentation Layer} (User Interface)\\
$\downarrow$\\
\textbf{2. Application Layer} (Business Logic)\\
$\downarrow$\\
\textbf{3. Data Layer} (Database and File Management)\\
$\downarrow$\\
\textbf{4. Hardware Interface Layer} (LiDAR, Camera, Sensors)
\end{tcolorbox}

\subsection{Module Descriptions}

\subsubsection{Presentation Layer}
\begin{itemize}
    \item \textbf{AR View Controller:} Main interface for scanning and visualization
    \item \textbf{Database Browser:} Interface for selecting and managing templates
    \item \textbf{Settings Panel:} Configuration options and calibration tools
    \item \textbf{Report Viewer:} Display inspection results and history
\end{itemize}

\subsubsection{Application Layer}
\begin{itemize}
    \item \textbf{Scan Manager:} Controls LiDAR scanning process
    \item \textbf{Recognition Engine:} Implements object identification algorithms
    \item \textbf{Comparison Engine:} Performs geometric analysis
    \item \textbf{Visualization Engine:} Generates AR overlays and heat maps
\end{itemize}

\subsubsection{Data Layer}
\begin{itemize}
    \item \textbf{Template Database:} Stores CAD models and specifications
    \item \textbf{Scan Cache:} Temporary storage for captured point clouds
    \item \textbf{Results Database:} Stores inspection history and results
    \item \textbf{Configuration Manager:} Handles user preferences and settings
\end{itemize}

\subsubsection{Hardware Interface Layer}
\begin{itemize}
    \item \textbf{ARKit Interface:} Access to LiDAR and motion tracking
    \item \textbf{Camera Interface:} RGB camera for visual feedback
    \item \textbf{File System Interface:} Storage management
\end{itemize}

\subsection{Data Flow}

\begin{enumerate}
    \item User launches application and selects inspection mode
    \item System activates LiDAR scanner and AR session
    \item User points device at target object
    \item LiDAR captures point cloud data in real-time
    \item Recognition engine identifies object type from templates
    \item System retrieves corresponding CAD model from database
    \item Comparison engine calculates dimensional deviations
    \item Visualization engine generates color-coded heat map
    \item AR overlay is rendered on physical object through device screen
    \item User reviews results and optionally saves report
\end{enumerate}

\section{Appendix A: Glossary of Terms}

\begin{description}
    \item[AR (Augmented Reality):] Technology that overlays digital information onto the real world
    \item[LiDAR (Light Detection and Ranging):] Laser-based 3D scanning technology
    \item[CAD (Computer-Aided Design):] Software for creating digital 3D models
    \item[Point Cloud:] Collection of 3D points representing a scanned surface
    \item[Mesh:] 3D surface model composed of vertices, edges, and faces
    \item[Heat Map:] Color-coded visualization showing variations in values
    \item[Template Database:] Collection of reference CAD models and specifications
    \item[Dimensional Deviation:] Difference between actual and design dimensions
    \item[CMM (Coordinate Measuring Machine):] Precision measurement device
    \item[ARKit:] Apple's framework for AR development on iOS
    \item[CoreML:] Apple's machine learning framework
    \item[FPS (Frames Per Second):] Measurement of rendering performance
\end{description}
